# wanted-project

원티드 오픈API 데이터를 재구성해서, 처음 구직하는 청년이 겪는 정보 비대칭과 파편화 문제를 줄여주는 개인화 대시보드입니다.

## 서비스 소개

- **타깃**: 경력 0년(완전 신입) 구직자 (1~2년차 주니어까지 포괄)
- **문제**: 신입 태그 공고인데 실제 요구 경력이 안 맞음, 채용 용어 장벽, 공고 홍수 속 마감 놓침, 지원 이력 파편화, 회사 정보(연봉/복지) 부족, 내 역량과 공고 요구사항의 갭 파악 어려움
- **데이터 소스**: Wanted OpenAPI v1/v2 + 자체 DB(회원/프로필/지원 상태)

기획 배경과 API 검토 과정은 [PLANNING.md](./PLANNING.md), 기능별 요구사항과 완료 기준은 [PRD.md](./PRD.md)에 정리되어 있습니다.

## 주요 기능

우선순위 순 (자세한 내용은 [PRD.md](./PRD.md) 참고):

1. 회원가입 & 프로필/포트폴리오
2. 신입 맞춤 필터 대시보드
3. 회사 정보(연봉/복지)
4. 용어 툴팁
5. 지원 여정 칸반보드
6. JD 매칭 진단 & 이력서 갭 체크리스트
7. 마감임박 지도 위젯
8. 신입 채용 트렌드 인사이트

경쟁 강도(지원자 수)와 합격률 예측은 API 검토 결과 스코프에서 제외했습니다 (사유는 PRD.md 6절 참고).

## 브랜치 전략

- `main`: 항상 배포 가능한 상태를 유지하는 기본 브랜치. 직접 push 금지, PR로만 병합합니다.
- 팀원별/기능별 브랜치에서 작업 후 `main`으로 Pull Request를 올려 리뷰 후 병합합니다.

브랜치 이름 예시:

```
git checkout -b feature/이름-작업내용
```

## 작업 흐름

1. `main`에서 최신 상태로 새 브랜치 생성
2. 작업 후 커밋 및 push
3. GitHub에서 `main`으로 Pull Request 생성
4. 팀원 리뷰 후 병합
