# 프로젝트 기획서

**서비스명**: 원티드 API 기반 대시보드
**페르소나**: 처음 구직을 시도하는 청년 (완전 신입이 메인 타깃, 1~2년차까지는 포괄)

## 문제상황 도출 및 가설검증 과정

처음 구직자의 불편함(경력요건 헷갈림, 용어 장벽, 공고 홍수, 마감 놓침, 지원이력 파편화, 회사정보 부족, 경쟁강도 모름)을 API로 충족 가능한지 검증한 결과, 아래 8개 기능을 전부 제작하기로 함.
(경쟁강도/합격률예측은 API 검토 결과 여전히 스코프 제외 — 사유는 하단 참고)

## 제작 예정 기능 (8개)

### 0. 회원가입 & 프로필/포트폴리오
- **문제**: 개인화 기능(관심 공고 저장, 지원 현황 관리, JD 매칭)이 동작하려면 사용자 식별과 프로필 정보가 있어야 함
- **해결**: 회원가입/로그인 + 프로필 등록 폼
  - 경력: 0~2년 범위로 좁게 설계 (메인 타깃은 완전 신입, 1~2년차까지 포괄)
  - 관심 직군/스킬 태그 등록
  - 포트폴리오(링크 또는 파일 업로드) 등록
- **비고**: 원티드 API가 주는 데이터가 아니라 순수 자체 구현(자체 DB + 인증). 4번(JD 매칭), 6번(칸반보드), 3번(지도위젯의 "지원한 곳" 마커)이 전부 이 기능의 사용자 상태를 사용함 → **다른 기능들의 선행 조건**

### 1. 신입 맞춤 필터 대시보드
- **문제**: 신입 태그 공고인데 실제 요구 경력이 1~3년인 경우가 많아 진짜 신입 가능 공고 구분이 안 됨
- **해결**: 공고 응답의 `annual_from`/`annual_to` 필드를 파싱해 "진짜 신입 가능" 공고만 필터링 + 직군/지역 조합 필터

### 2. 용어 툴팁
- **문제**: 공고 속 기술 스택/직무 용어를 몰라서 지원 여부 판단이 어려움
- **해결**: 공고의 `skill_tags`에 툴팁으로 쉬운 설명 제공 (API에는 `{id, title}`만 있어 설명 데이터는 별도 구축 필요)

### 3. 마감임박 지도 위젯 (마감임박 위젯 + 지도 마커 뷰 + 통근 반경 필터 통합)
- **문제**:
  - 여러 공고를 보다가 마감일을 놓쳐 지원 기회를 날림
  - 공고가 텍스트 리스트로만 나오면 위치 감이 안 오고, 통근 가능한 곳인지 한눈에 안 보임
  - "신입 가능 공고" 중에서도 통근 가능한 곳인지 따로 알아봐야 함
- **해결**: 마감일(`due_time`) 기준 D-day 정렬 리스트를 지도 뷰와 결합. 외부 지도 API(카카오맵/구글맵) 위에 마커로 표시:
  - 마감임박 공고 (마감일 기준 색상 구분)
  - 내가 지원한 곳 (0번 프로필/6번 칸반보드의 사용자 상태 필요)
  - 관심 등록한 공고가 올라온 곳
  - 사용자 현재 위치 기준 반경/통근시간 필터로 좁혀보기
- **API 근거**: `/jobs/{job_id}` 상세 응답의 `address.geo_location`에 위경도가 이미 포함되어 있어 별도 geocoding 없이 마커 좌표 확보 가능
- **트레이드오프**: 목록(`/jobs`) 응답에는 주소 텍스트만 있고 `geo_location`이 없어서, 여러 공고를 한번에 지도에 찍으려면 공고별 상세 API 추가 호출 또는 주소 텍스트 자체 geocoding이 필요함

### 4. JD 매칭 진단 & 이력서 갭 체크리스트 (JD 매칭 진단 + 갭 체크리스트 통합)
- **문제**:
  - 내 이력서/역량이 이 공고 기준에 맞는지 판단이 어려움
  - 매칭률만 보여주면 그래서 뭘 해야 하는지 액션으로 이어지지 않음
- **해결**: 공고의 `skill_tags`·`detail.preferred_points`(우대사항)와 내 프로필(0번에서 등록한 관심 태그/포트폴리오)을 비교해 매칭률/갭을 보여주고, 갭으로 나온 스킬을 "이 스킬 채우면 매칭률 오름" 형태의 온보딩형 체크리스트로 제공

### 5. 회사 정보(연봉/복지)
- **문제**: 지원 전에 이 회사가 어떤 곳인지(연봉 수준, 복지) 알기 어려움
- **해결**:
  - **연봉**: `/insight/company?biz_number=` 호출로 `averageSalary`(평균연봉), `hiredSalary`(신규입사자 평균연봉) 등 통계 데이터 제공. `biz_number`는 공고 조회 시 받는 `company.registration_number`로 바로 확보 가능
  - **복지**: 공고 상세 `detail.benefits`(텍스트) + `attraction_tags`(매력태그, 예: "리프레시휴가") 구조화 태그로 제공
- **주의**: `/insight/company`는 별도 백엔드 연동으로 보여 응답 안정성(503 케이스 존재)·쿼터를 실제 계정으로 확인 필요

### 6. 지원 여정 칸반보드
- **문제**: 여러 곳에 지원하다 보면 어디에 지원했는지, 어느 단계인지 파편화되어 관리가 안 됨
- **해결**: 관심 → 준비중 → 지원함 → 결과대기 컬럼의 칸반보드. 공고 스냅샷(회사명/마감일/스킬태그)은 API에서 채우고, 진행 단계는 0번 프로필 기능의 사용자 계정에 자체 저장
- **주의**: 원티드 API에는 "내가 지원한 공고의 진행 상태" 조회가 없음(그건 채용사용 ATS API라 구직자는 접근 불가) → 상태 관리는 반드시 자체 DB로 구현

### 7. 신입 채용 트렌드 인사이트
- **문제**: 처음 구직자는 뭘 준비해야 할지 기준이 없음
- **해결**: `/tags/skills` 전체 목록 대비, 신입 태그 공고들의 `skill_tags` 빈도를 집계해 "신입이 많이 요구받는 스킬 TOP10" 같은 통계 제공

## 우선순위

**0번(회원가입/프로필)과 1번(신입 맞춤 필터)을 최우선 병렬 작업으로 시작.** 하나는 계정/DB 인프라, 하나는 API 연동 코어라 팀원 두 명이 동시에 시작할 수 있고, 이후 모든 기능이 이 둘에 의존함.

1. **0. 회원가입 & 프로필/포트폴리오** — 인프라, 다른 기능들의 선행 조건
2. **1. 신입 맞춤 필터 대시보드** — API 연동 코어, 공고 데이터 자체가 여기서 시작됨
3. **5. 회사 정보(연봉/복지)** — 계정 상태 없이 바로 붙는 고임팩트 기능
4. **2. 용어 툴팁** — 콘텐츠 큐레이션이 관건이라 개발과 병렬 가능
5. **6. 지원 여정 칸반보드** — 0번의 사용자 상태를 실제로 활용하는 첫 기능
6. **4. JD 매칭 진단 & 갭 체크리스트** — 0번의 프로필/포트폴리오 데이터를 활용
7. **3. 마감임박 지도 위젯** — 지도 API + 6번의 "지원한 곳" 데이터까지 합쳐야 해서 통합도 가장 높음
8. **7. 신입 채용 트렌드 인사이트** — 1번만 있으면 언제든 붙일 수 있는 통계 기능, 시간 남으면 추가

## 제외된 기능 (API 재검토 완료)

- **경쟁 강도(지원자 수)**: 공고 단위로 다른 지원자 수를 노출하는 필드가 API 전체에 없음. `/stat/application/summary`는 "우리 client_id로 접수된 지원" 집계용이라 구직자 관점에는 사용 불가 → **확정 제외**
- **합격률 예측**: `/ai/pass/text-prediction/async` 엔드포인트가 정확히 이 기능이지만, "별도 계약 후 사용 가능(문의: team-ml@wantedlab.com)"인 유료 상품이라 현재 크레덴셜로는 사용 불가. 비동기 콜백 구조라 서버 인프라도 추가 필요 → **MVP 스코프 제외** (데이터 부재가 아니라 유료 계약 이슈)

## Phase 2 아이디어 (스코프 밖, 추후 검토)

- **기업용 입력 기능** ("진짜 신입" 인증 배지, 탈락자 피드백 요약, 채용 프로세스 투명성 등): 원티드 API의 기업용 엔드포인트(`/ats/position`, `/recruit-company/*`)는 우리 `client_id`/`client_secret`가 아니라 별도의 `x-wanted-dashboard-service-key`로 인증하는 다른 상품이라, 이 기능들은 원티드 API로 못 가져오고 자체 기업 계정/입력 폼을 처음부터 만들어야 함 → 구직자 대시보드에서 양면 플랫폼으로 스코프가 커지는 결정이라 별도 검토 필요
